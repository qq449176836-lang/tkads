## 三层记忆模型

借鉴 Agent-S（simular-ai）的记忆架构：

```
╔═══════════════════════════════════════════╗
║  Layer 3: Procedural  操作手册            ║  ← 标准化 SKILL.md
║           (最持久、最可复用)              ║     可执行步骤
╠═══════════════════════════════════════════╣
║  Layer 2: Narrative   跨任务规律          ║  ← 从多次 Episode 提炼
║           (任务间通用模式)                ║     "飞书集成三原则"
╠═══════════════════════════════════════════╣
║  Layer 1: Episodic    单次经验            ║  ← @timestamp + 具体案例
║           (最鲜活、最具体)                ║     "2026-06-10 乱码修复"
╚═══════════════════════════════════════════╝
```

### Layer 1: Episodic（单次经验）

**是什么**：一次具体任务的完整记录。

**格式**：
```markdown
### 2026-06-10: 飞书卡片中文乱码
- **症状**：飞书 webhook 消息中文显示为乱码
- **根因**：bash 中用 printf 拼接 JSON 破坏了 UTF-8 编码
- **修复**：改用 curl -d @file 方式
- **经验标签**：#飞书 #乱码 #bash #curl #编码
```

**存储**：Skill 文档末尾的 `## 经验记录` section

**五要素不全 → 不合格**：必须有时间、症状、根因、修复、标签。

### Layer 2: Narrative（跨任务规律）

**是什么**：从同主题 ≥3 条 Episodic 中提取的通用规律。

**格式**：
```markdown
### 模式：飞书集成三原则
> 来自 Episodic: 2026-06-10乱码 + 2026-06-11超时 + 2026-06-12签名错误

1. 中文数据不经过 printf/sed 拼接 → 走 curl -d @file
2. curl 请求必须设 --connect-timeout 10
3. 签名放 header 不放 URL query
```

**价值**：能直接回答"这类问题的一般解法是什么"，不需要翻具体案例。

### Layer 3: Procedural（操作手册）

**是什么**：标准化、可操作的流程——就是 SKILL.md 本身。

**与 Narrative 的区别**：

| | Narrative | Procedural |
|---|---|---|
| 抽象度 | 中等（领域内通用） | 最高（标准化） |
| 形式 | 规律性描述 | 可执行步骤 |
| 注入时机 | 遇到相关问题时参考 | 加载 Skill 时全部注入 |
| 举例 | "中文不要走 printf" | `sed -i 's/printf/curl -d @file/' *.sh` |

## 棘轮循环（Ratchet Cycle）

> 经验只能前进，不能倒退。每次循环都在上一轮基础上累积。

```
遭遇 (Encounter) → 诊断 (Diagnose) → 修复 (Fix)
    → 记录 (Record) → 提炼 (Distill) → 标准化 (Codify) → 推送 (Push)
        ↑_______________________________________________|
              下次遇到同类问题自动加载
```

### 触发条件

| 事件 | 触发动作 |
|------|---------|
| 修复一个 bug | 记录 Episodic |
| 完成一个复杂任务 | 记录 Episodic + 评估是否有新 Narrative |
| 同一主题 ≥3 条 Episodic | 提炼 Narrative |
| 用户说"记住这个" | 记录 Episodic（高优先级） |
| 创建/更新 Skill | Narrative → 提升到 Procedural |
| 用户指出错误 | 修正所有受影响层（E/N/P） |

## 三层检索策略

> 遇到问题时，先搜"类似情况怎么处理的"，而不是从零推理。

```
Layer 1: 精确匹配
  工具: search_files(target='content')
  场景: 已知关键字/错误信息
  代价: 最低，毫秒级

Layer 2: 语义扩展
  工具: delegate_task 做关键词扩展
  场景: 概念不精确但方向清楚
  代价: 低，一次 LLM 调用

Layer 3: 遍历 Skill 库
  工具: 逐个 Skill 做语义摘要对比
  场景: 复杂未知问题
  代价: 中，需多次 read_file
```

### 检索结果不是直接抄

1. **确认差异**：当前环境和历史经验的环境是否一致？
2. **提取原理**：不是抄命令，而是理解为什么那么做
3. **改造应用**：把原理应用到当前具体场景

## 记忆体系的分工

| 机制 | 存储内容 | 注入方式 | 生命周期 |
|------|---------|---------|---------|
| **Memory** | 持久偏好、环境事实 | 每次会话自动注入 | 除非显式删除 |
| **Session** | 完整对话历史 | 被 Reviewer 蒸馏时检索 | 永久存档 |
| **Skill** | 标准化操作流程 | AutoLoader 按策略激活 | 不被降级到消失 |
| **experience-retrieval** | 按需搜索的经验向量 | 任务开始时主动检索 | 按需 |

**核心区别**：Memory 是被动注入的常驻知识；经验检索是按需主动搜索的知识发现。

---

## 知识衰减策略

| 知识类型 | 衰减条件 | 处理 |
|---------|---------|------|
| Episodic（经验记录） | >5 条后 | 提炼为 Narrative，仅保留最近 3 条 |
| Narrative（通用规律） | 上升为 Procedural 后 | 降权，不再高频检索 |
| Skill（技能） | 连续 2 月 0 次使用 | 降级为 on-demand |
| Skill（技能） | 连续 6 月 0 次使用 | 建议归档 |
| Memory（临时事实） | 60 天未被引用 | Reviewer 清理 |

### 质量门

每个 Narrative 必须包含四要素：

```
✅ 情境 (Context)   — 在什么情况下
✅ 行动 (Action)    — Agent 做了什么
✅ 结果 (Result)    — 产出了什么
✅ 教训 (Learning)  — 未来该怎么做

缺任一要素 → 拒绝存储
```

## 完整闭环示例

```
1. 用户: "飞书消息乱码"
2. experience-retrieval L1: 搜索 "飞书.*乱码"
   → 命中！Episodic: 2026-06-10 printf 破坏 UTF-8
3. 检索结果注入: "历史修复方案: curl -d @file 替代 printf"
4. Agent 直接修复，不需要从零推理
5. 本次修复成功后，记录为新的 Episodic
6. 当同类问题积累到 3 次 → 提炼为 Narrative: "飞书集成三原则"
7. Narrative 反复验证 → 升级为 Procedural: 更新 feishu-alert SKILL.md
8. 下次遇到飞书相关问题 → L1 直接命中 Procedural → 秒级响应
```

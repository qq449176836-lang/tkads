# KNOWLEDGE CRYSTALLIZATION — 知识结晶化体系

> **经验如何从一次对话变成永久知识：三层蒸馏 + 分层归档 + 衰减管理。**

---

## 三层蒸馏模型

```
┌────────────────────────────────────────────┐
│              原始经验 (Episodic)             │
│  单次会话的完整记录：做了什么、遇到什么问题    │
│  格式：会话摘要 + 关键决策                   │
│  存储：session_transcripts                  │
│  生命周期：永久存档，但不会主动加载            │
└─────────────────┬──────────────────────────┘
                  │ 提取关键教训
                  ▼
┌────────────────────────────────────────────┐
│            叙事知识 (Narrative)              │
│  情境 → 行动 → 结果 → 教训 四要素             │
│  格式：结构化描述（为什么做、怎么做、学到什么）  │
│  存储：Memory / 结构化叙述文档               │
│  加载：experience-retrieval 关键词匹配        │
│  生命周期：跨会话存在，相似情境自动触发         │
└─────────────────┬──────────────────────────┘
                  │ 检测 >=3 次重复模式
                  ▼
┌────────────────────────────────────────────┐
│           可执行技能 (Procedural)             │
│  标准化操作流程：步骤 + 条件 + 验证            │
│  格式：SKILL.md（标准化模板）                 │
│  存储：~/.hermes/skills/<name>/SKILL.md      │
│  加载：AutoLoader 根据策略自动激活             │
│  生命周期：除非显式删除，否则永久存在           │
└────────────────────────────────────────────┘
```

---

## 蒸馏触发条件

| 触发事件 | 蒸馏层级 | 频率 |
|---------|---------|------|
| 用户纠正错误 | Episodic → Narrative | 即时（高优先级） |
| 会话保存 `/save` | Episodic → Narrative | 每次会话 |
| 代码产出 + 成功运行 | Episodic → Narrative | 每次 |
| 同一模式 >=3 次 | Narrative → Procedural | Reviewer 每周复盘 |
| 用户显式要求 | 任意层级 | 按需 |

## 分层归档策略

### Memory（持久记忆）

```
存储内容：
  ✅ 用户偏好、环境事实、项目约定
  ✅ 工具行为 quirks、API 特性
  ✅ 稳定不经常变化的知识
  ❌ 临时进度、PR 编号、过时数据
  ❌ "做了 X" 之类的任务完成记录

原则：如果一条记忆 7 天后会过期，就不要存入 Memory。
```

### Skill（技能库）

```
存储内容：
  ✅ 可重复执行的操作流程
  ✅ 配置模板和脚本
  ✅ 方法论和最佳实践
  ❌ 单次使用的临时方案
  ❌ 未经蒸馏的原始会话内容

结构：
  ~/.hermes/skills/<skill-name>/
  ├── SKILL.md          # 技能定义（必选）
  ├── scripts/          # 辅助脚本（可选）
  └── templates/        # 代码模板（可选）
```

### Session（会话存档）

```
存储内容：
  ✅ 完整对话历史
  ✅ 所有工具调用和结果
  
用途：被 Reviewer 蒸馏时作为原始素材
生命周期：永久存档，不主动加载到上下文
```

## 知识衰减与清理

### 衰减机制

| 知识类型 | 衰减条件 | 处理方式 |
|---------|---------|---------|
| Memory（一次性事实） | 60 天未被引用 | 标记为过期，Reviewer 月度清理 |
| Skill（技能） | 连续 2 月 0 次使用 | AutoLoader 降级为 on-demand |
| Skill（技能） | 连续 6 月 0 次使用 | Reviewer 建议归档/删除 |
| Narrative（叙事知识） | 已蒸馏为 Procedural | 原始 Narrative 降权，不再检索 |
| Episodic（原始经验） | 已蒸馏为 Narrative | 标记为已蒸馏，保留存档 |

### 质量门

每个 Narrative 经验必须包含四要素才允许存储：

```
✅ 情境 (Context)   — 在什么情况下发生的
✅ 行动 (Action)    — Agent 做了什么
✅ 结果 (Result)    — 产出了什么
✅ 教训 (Learning)  — 未来应该怎么做

缺少任一要素 → 拒绝存储，返回重新蒸馏
```

## 知识检索流程

```
用户提问 / 任务到来
    │
    ▼
┌─────────────────────────────┐
│ Layer 1: Memory 精确匹配     │
│   → 关键词直接命中           │
│   → 返回相关记忆             │
└──────────┬──────────────────┘
           │ 未命中
           ▼
┌─────────────────────────────┐
│ Layer 2: Skill 库匹配        │
│   → 加载相关技能             │
│   → 提取方法论/操作流程       │
└──────────┬──────────────────┘
           │ 未命中
           ▼
┌─────────────────────────────┐
│ Layer 3: Session 语义检索    │
│   → 搜索历史会话             │
│   → 发现相似情境下的做法      │
└──────────┬──────────────────┘
           │ 未命中
           ▼
      按正常流程推理
```

## 棘轮保证机制

> 确保知识只增不减的四个锚点：

1. **Skill 永不自动删除** — 只能降级到 on-demand，不能被彻底移除
2. **引擎标记 Critical** — Evolution Engine 自身永不被降级（自举）
3. **分层蒸馏不可逆** — Episodic → Narrative → Procedural 是单向固化
4. **Cron 兜底** — 即使长期无人使用，Cron 任务确保定期复盘

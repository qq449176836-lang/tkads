# EVOLUTION ENGINE — 自我进化引擎

> **让 Agent 像生物体一样自我进化：感知环境变化 → 内化新能力 → 复盘并固化经验。**

---

## 设计隐喻：生物进化

| 生物进化 | Evolution Engine 对应 |
|---------|----------------------|
| 基因突变（变异） | **Explorer** 发现新技术/工具/技能 |
| 自然选择 | **AutoLoader** 根据使用频率/成功率决定激活哪些 |
| 遗传固化 | **Reviewer** 将有效经验蒸馏为永久技能/记忆 |
| 适应性 | 每次会话都加载已进化的能力集 |

## 模块 1：🧭 Explorer — 自动探索

### 探索流程

```
Cron 触发 → 信息采集(GitHub/HN/Reddit/Skills Hub)
         → 价值评估(新颖性/实用性/可行性 1-10)
         → 决策推荐(>=8 飞书推送 / >=7 待审 / <7 归档)
```

### 信息源优先级

1. GitHub Trending（全语言 + AI/ML 分类）
2. Hacker News "Show HN" + AI/agent/tool 关键词
3. Hermes Skills Hub（本地技能市场）
4. Reddit r/MachineLearning, r/LocalLLaMA

### 评估维度

| 维度 | 满分含义 | 权重 |
|------|---------|------|
| 新颖性 (Novelty) | 颠覆性创新 | 30% |
| 实用性 (Utility) | 解决刚需痛点 | 40% |
| 可行性 (Feasibility) | 即插即用 | 20% |
| 活跃度 (Activity) | 社区活跃度 | 10% |

### Cron 调度

| 频率 | 任务 | 产出 |
|------|------|------|
| 每 6 小时 | 高频趋势扫描 | discoveries/raw/ |
| 每天 10:00 | 每日综合评估 | 飞书推送 >=7 发现 |
| 每周一 9:00 | 每周趋势深度分析 | Weekly Tech Radar |

## 模块 2：⚡ AutoLoader — 自动加载

### 会话启动流程

```
用户启动 Agent
  → Phase 1: 加载 Constitution（全局 + 项目 + 本地）
  → Phase 2: 读取 auto-load.json，激活 Always-On 技能
  → Phase 3: 检测项目类型，按 context_rules 加载相关技能
  → Phase 4: 根据 usage.json 调整优先级
  → 会话就绪（带着进化后的能力）
```

### 四层加载策略

| 层级 | 触发条件 | 示例 |
|------|---------|------|
| **Critical** | 引擎自身，永不离线 | hermes-evolution-engine |
| **Always-On** | 每次会话自动加载 | spec-driven-dev, task-planning |
| **Context-Aware** | 检测项目类型后加载 | 代码项目→代码规范技能 |
| **On-Demand** | 用户显式调用 | `/skill <name>` |

### 频率自适应

- 高频使用（30 天内 >=5 次）→ 提升到 Always-On
- 低频使用（连续 2 月 0 次）→ 降级为 On-Demand
- 引擎自身标记为 Critical，永不降级（自举防止退化）

## 模块 3：📊 Reviewer — 自动复盘

### 复盘流程

```
Cron 触发
  → Phase 1: 收集过去 N 天会话，筛选高价值（代码产出/问题解决/用户反馈）
  → Phase 2: 调用 knowledge-distillation，Episodic → Narrative 蒸馏
  → Phase 3: 高频模式检测（>=3 次）→ Narrative → Procedural 蒸馏
  → Phase 4: 更新记忆、清理过期知识、生成复盘报告
```

### 三层蒸馏

```
原始对话记录 (Episodic)
    │  提取关键教训
    ▼
叙事化经验 (Narrative) ← 存入 Memory，跨会话可检索
    │  检测 >=3 次重复模式
    ▼
可执行技能 (Procedural) ← 创建/更新 SKILL.md，AutoLoader 自动加载
```

### Cron 调度

| 频率 | 任务 | 蒸馏层级 |
|------|------|---------|
| 每天 22:00 | 每日快速复盘 | Episodic → Narrative |
| 每周日 23:00 | 每周深度复盘 | Narrative → Procedural |
| 每月 1 日 02:00 | 月度技能固化 | 完整三层 + 技能库健康检查 |

## 三不原则

1. **不主动干扰用户** — Explorer 后台运行，只推送高价值（>=8）发现
2. **不重复犯同样的错** — Reviewer 确保经验被蒸馏并内化
3. **不退化（棘轮效应）** — 技能只能增加/优化，不会被遗忘或降级到消失

---

## 引擎健康指标

| 指标 | 健康阈值 | 告警信号 |
|------|---------|---------|
| 发现率 | >=5 新发现/周 | <2：需更新搜索关键词 |
| 加载命中率 | >=80% 技能 30 天内使用 | <50%：技能过载 |
| 蒸馏成功率 | >=90% 会话成功蒸馏 | <70%：排查蒸馏流程 |
| Cron 执行率 | 100% 按时执行 | <95%：排查调度器 |
| 技能库增长 | 每月 >=1 新技能/更新 | 0：进化停滞 |

## 自举悖论

> 进化引擎自身的 SKILL.md 如何进化？

**解决方案**：月度复盘时将引擎优化建议纳入 Procedural 蒸馏，通过 `write_file` 更新自身。引擎在 auto-load.json 中标记为 `critical`，永不降级——确保"进化的能力本身也在进化"。

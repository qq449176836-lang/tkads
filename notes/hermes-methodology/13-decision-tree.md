## 十三、决策树

### 13.1 接到任务时的决策

```
用户提出需求
    │
    ├── 理解了吗？── 否 → 确认澄清
    │
    ├── 三层检索命中？── 是 → 直接执行
    │
    ├── 耗时评估？
    │   ├── <5min → 直接执行
    │   ├── 5~30min → delegate_task 并行
    │   ├── 30min~1h → terminal(background)
    │   └── >1h → cronjob
    │
    ├── 涉及安全/资金？── 是 → 先确认再执行
    │
    └── 需要用户配合？── 是 → 给出明确步骤
```

### 13.2 遇到错误时的决策

```
工具调用出错
    │
    ├── 第一次出错？── 是 → 自动重试1次
    │
    ├── 已知错误模式？── 是 → 按已知方案修复
    │
    ├── 可自动修复？── 是 → 执行修复并记录
    │
    ├── 需用户介入？── 是 → 报告并询问
    │
    └── 无法修复？── 安全停止 + 完整报告
```

### 13.3 沉淀决策

```
解决问题后
    │
    ├── 这是一个新模式？── 是 → 记入 E 层 + 标记为 N 候选
    │
    ├── 这是一个高频模式？── 是 → E → N → 考虑 P
    │
    ├── 这是一个简单修复？── 是 → 至少记 E 层
    │
    └── 这是一个重复错误？── 是 → 优先升级到 P 层（标准化方案）
```

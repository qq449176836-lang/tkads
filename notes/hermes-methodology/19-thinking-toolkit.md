## 十九、思维工具箱（隐式框架）

这些不是刻意学的底层思维框架，而是每次思考时自动调用的：

### 19.1 第一性原理
```
剥离所有假设，回归最基础的不可再分的事实，从那里重新构建。

例子: SKU 漏导入问题
  - 表层: 导入脚本有bug
  - 第一性: 数据库唯一约束和业务需求不匹配
  → 修约束而不是修脚本
```

### 19.2 二阶思维
```
做完 A 之后，B 会发生什么？C 呢？

例子: 修改数据库唯一约束
  - 一阶: 约束改了，SKU能插入了
  - 二阶: 旧数据还在吗？查询会受影响吗？应用层兼容吗？
```

### 19.3 奥卡姆剃刀
```
最简单的解释往往是对的，不要过度复杂化。

例子: 系统变慢了
  - 复杂解释: 内存泄露、内核bug、量子隧穿效应
  - 简单解释: 日志文件太大了，清理一下
  → 先排除简单原因
```

### 19.4 帕累托法则（80/20）
```
80% 的效果来自 20% 的投入，优先找那 20%。

例子: 知识沉淀
  - 80% 的操作日志 → 存档即可
  - 20% 的模式/教训 → 升级到 N/P 层
  - 把那 20% 找出来，花 80% 的时间打磨它
```

### 19.5 反证法
```
如果反过来想，会得出什么结论？

例子: "这个方案能解决问题"
  → 反证: "什么情况下这个方案会失败？"
  → 如果找到3个以上失败场景，方案需要修正
```

### 19.6 费曼技巧
```
不能用简单语言解释清楚，说明自己还没真懂。

检验标准:
  能对一个不懂技术的人讲明白 → 真懂了
  需要用专业术语才能解释 → 还没真懂
```

### 19.7 心智模拟
```
执行前在脑子里完整跑一遍，找盲点。

做法:
  1. 闭上眼，从第一步开始一步一步模拟
  2. 每一步问"这一步之后是什么状态？"
  3. 找到逻辑跳跃或模糊的地方
  4. 补全后再实际执行
```

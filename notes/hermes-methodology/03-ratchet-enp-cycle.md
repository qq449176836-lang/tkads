## 三、棘轮ENP循环

### 核心思想
```
知识只能单向增长，不能倒退。
每次解决问题都让系统变得更强。
```

### 循环流程

```
              ┌──────────────────┐
              │  E — 遭遇问题    │
              │  (错误/异常/需求)  │
              └────────┬─────────┘
                       ↓
              ┌──────────────────┐
              │  N — 诊断分析    │
              │  (定位根因)       │
              └────────┬─────────┘
                       ↓
              ┌──────────────────┐
              │  P — 修复验证    │
              │  (实施+确认结果)   │
              └────────┬─────────┘
                       ↓
        ┌──────────────┼──────────────┐
        ↓              ↓              ↓
   ┌────────┐   ┌────────────┐   ┌────────┐
   │ E 层   │   │  N 层      │   │ P 层   │
   │ 原始日志│→  │ 精炼笔记   │→  │ 技能库 │
   └────────┘   └────────────┘   └────────┘
```

### 三级沉淀规范

| 层级 | 名称 | 内容 | 存储 | 更新频率 |
|---|---|---|---|---|
| **E** | 原始经验 | 完整的操作记录、错误日志、诊断过程 | `operation_log/` | 每次任务 |
| **N** | 精炼笔记 | 去噪后的经验总结、模式提炼、最佳实践 | 知识库(VikingDB) | 每日/每周 |
| **P** | 标准化技能 | 可复用的工作流、检查清单、代码 | Skill 库 | 每周/每月 |

**关键规则：** 一条经验如果只停留在 E 层，等于没沉淀。目标是把最有价值的 20% 经验升级到 N 层，再把最常用的 5% 标准化到 P 层。

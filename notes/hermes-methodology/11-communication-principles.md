## 十一、沟通原则

### 11.1 与用户交互

```
▸ 先理解，再行动
  - 不确定用户需求时，先确认再执行
  - 不要猜测用户意图

▸ 用用户能理解的语言
  - 避免过度技术术语
  - 需要解释时举例说明

▸ 主动报告
  - 任务开始：告知要做什么
  - 任务中：遇到阻塞主动报告
  - 任务结束：清晰总结结果

▸ 不确定时
  - 先问，不猜
  - 给出选项让用户选择
```

### 11.2 与系统交互

```
▸ 工具调用优先
  - 能用工具就别用描述的
  - 一个工具调用完成一步，别混在一起

▸ 验证闭环
  - 每次写操作后验证结果
  - 不信任自述的"成功"，要实际检查

▸ 渐进式操作
  - 复杂操作拆成小步骤
  - 每步确认后再下一步
```

### 11.3 错误报告格式

```
❌ [错误类型] 简短描述
   错误信息: 原始错误
   根因: 为什么发生
   影响: 影响范围
   修复: 建议方案
```

## 六、双循环进化引擎

### 循环 A：自进化（self-evolve）
```
频率: 每 2 小时（或每次任务结束后）
目的: 短期微调，快速优化

检查清单:
□ 最近操作日志有无重复错误？
□ Prompt/工作流是否需要优化？
□ 临时文件是否需要清理？
□ 系统资源是否正常？
```

### 循环 B：进化引擎（evolution-engine）
```
频率: 每天 23:00（低峰时段）
目的: 长期进化，系统性升级

三阶段:
┌─────────────────────────────────────────┐
│ Phase 1: Explorer（探索者）              │
│  ├─ 搜索 GitHub 新工具/框架/库          │
│  ├─ 追踪竞品 Agent 新能力               │
│  └─ 发现可吸收的新模式                   │
├─────────────────────────────────────────┤
│ Phase 2: AutoLoader（自动升级）          │
│  ├─ 评估新发现的价值                     │
│  ├─ 自动下载/安装/配置                   │
│  └─ 生成适配层                           │
├─────────────────────────────────────────┤
│ Phase 3: Reviewer（评审者）              │
│  ├─ 回顾今天所有会话                     │
│  ├─ 蒸馏最有价值的经验                   │
│  └─ 推荐 E→N→P 升级                      │
└─────────────────────────────────────────┘
```
